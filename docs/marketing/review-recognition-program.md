# Review Recognition Program

**Category:** Marketing - Operations
**Last Updated:** July 23, 2026
**Status:** Active Draft for Leadership Approval

> **Brand Reference:** See [Brand Constants](../branding/brand-constants.md) for canonical brand values.
>
> **Brand Congruency:** This program must remain relationship-first, factual, trust-preserving, and compliant with platform policies.

## Purpose

Create a repeatable, compliant system to increase authentic Google reviews while recognizing team members who are named by project stakeholders.

## Program Summary

- Internal name: Field Recognition Review Program.
- Review destination: [Google Review Link](https://g.page/r/CVdv3YZLzJvdEAE/review).
- Recognition amount: 25 dollars Cabela's gift card per qualifying review.
- Frequency: One reward per unique reviewer per completed project.

## Compliance Rules

1. Never request only positive or five-star reviews.
2. Always request honest feedback.
3. Never require specific wording from reviewers.
4. Never gate service, warranty, or support based on review submission.
5. Never ask employees to post reviews for MH Construction.

## Qualification Rules

A review qualifies when all items below are true:

1. The review is publicly visible on the MH Construction Google profile.
2. The reviewer appears to be a real stakeholder contact (client, owner representative, facility contact, or similar).
3. The review references a real project interaction.
4. The review names at least one MH employee.
5. The reviewer has not already generated a rewarded review for the same project.

## Disqualification Rules

- Duplicate reviews from the same person for the same project.
- Reviews that appear spam-like, fake, copied, or unrelated.
- Reviews posted by employees, immediate family, or vendors acting on behalf of MH.
- Reviews removed by Google before verification closeout.

## Reward Rules

1. Named employee receives one 25 dollars Cabela's gift card per qualifying review.
2. If multiple employees are named, leadership assigns one recipient using the project lead decision rule unless local policy specifies split rewards.
3. Finance or admin issues gift cards weekly.
4. Recommended cap: 5 rewarded reviews per employee per month unless leadership approves an exception.

## Operational Workflow

### 1) Trigger

- Trigger event: Project substantial completion or closeout milestone.
- Owner: Project Manager.
- Timing: Request 3 to 7 days after closeout.

### 2) Outreach

- Send one email request.
- Send one SMS follow-up if no review is posted after 7 days.
- Optional neutral verbal reminder during final handoff.

### 3) Verification

- Marketing or admin verifies qualification against this SOP.
- Log review URL, reviewer name, project, named employee, and verification date in the tracker.

### 4) Reward Fulfillment

- Admin issues card and logs fulfillment date and card reference ID.
- Notify employee and PM in weekly recognition summary.

### 5) Audit

- Monthly audit of duplicates, exceptions, and unresolved entries.
- Retain records for 12 months.

## Team Script Guardrails

Approved ask:

"Thank you for partnering with MH Construction. If you are open to it, we would value your honest Google review about your project experience."

Do not say:

- "Please leave us a five-star review."
- "Mention my name and I will get a bonus."
- "Only leave a review if you were happy."

## KPIs

- Review request-to-review conversion rate.
- Net new monthly reviews.
- Named-employee mention rate.
- Median days from closeout to posted review.
- Cost per qualifying review.

## 30-Day Launch Plan

1. Week 1: Finalize policy, brief PMs and superintendents, and publish templates.
2. Week 2: Start closeout-triggered email plus SMS sequence.
3. Week 3: Begin weekly recognition and fulfillment cadence.
4. Week 4: Review conversion metrics and optimize copy/timing.

## Ownership

- Program owner: Marketing lead.
- Execution owners: Project Managers plus admin support.
- Compliance owner: Leadership review monthly.
