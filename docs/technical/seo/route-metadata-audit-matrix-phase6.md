# Route Metadata Audit Matrix (Phase 6.1)

**Category:** Technical - SEO Audit
**Last Updated:** July 19, 2026
**Scope:** Public website routes through Phase 5 additions

## Purpose

This matrix records page-level metadata ownership and behavior for all current public route families, using the canonical route inventory from `apps/website/src/lib/seo/route-manifest.ts` and route class policy from `apps/website/config/seo/route-indexing-policy.json`.

## Canonical Language and URL Rules

- Canonical URL strategy is unprefixed and single-URL per route.
- `en` and `es` are cookie-selected on shared unprefixed URLs.
- Do not emit `/en` canonicals.
- `es` alternates are emitted only where route manifest locale parity allows.

### Phase 6.2 Guardrail Notes

- Phase 6.2 additions reference the global organization entity by `@id` instead of re-declaring full organization objects in new route schemas.
- New page-type schemas for `/events/[slug]`, `/services/[slug]`, and `/news` avoid unsupported claims such as ratings, offers, pricing, attendance counts, or project values.
- JSON-LD serialization for events hub and new route-level schemas is emitted through the existing `StructuredData` renderer for safer script serialization.
