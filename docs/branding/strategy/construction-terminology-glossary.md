# Construction Terminology Glossary

**Category:** Branding - Strategy
**Last Updated:** July 12, 2026
**Status:** Canonical Reference

## Purpose

This glossary defines the approved MH Construction language for construction management concepts.
It keeps the brand aligned with military-themed construction vocabulary while still mapping cleanly to
PMBOK 6 concepts and common industry expectations.

Use this glossary when writing or reviewing copy for website pages, dashboard labels, handbook language,
manuals, proposals, or implementation docs.

## How To Use

1. Use the MH term family first in public-facing copy.
2. Use the PMBOK term only when a training, compliance, or technical reference explicitly requires it.
3. Do not mix civilian consulting language into public copy when an MH term already covers the concept.
4. Keep external audiences as Project Stakeholders and delivery-side collaborators as Trade Partners.

## Core Glossary

| MH Term Family                               | PMBOK 6 / Construction Concept         | Preferred Usage                                                  |
| -------------------------------------------- | -------------------------------------- | ---------------------------------------------------------------- |
| Project Stakeholders                         | Stakeholder management                 | external review audiences, owners, architects, banks, insurers   |
| Dashboard (Staff Hub)                        | Project controls / internal operations | employee workflow, task management, internal routing             |
| MISH Safety & Health Program (Safety Manual) | Safety management / compliance program | safety manuals, forms, handbooks, compliance docs                |
| Scope architecture                           | Scope management                       | front-end planning, deliverable boundaries, scope-fit review     |
| Sequence logic                               | Schedule management                    | milestone planning, work sequencing, execution cadence           |
| Budget clarity                               | Cost management                        | pricing visibility, budget control, estimate confidence          |
| Quality checkpoints                          | Quality management                     | inspection discipline, first-time-right execution                |
| Crew readiness                               | Resource management                    | staffing, capacity, field readiness, crew allocation             |
| Briefings and handoffs                       | Communications management              | updates, status calls, transition points, turnover communication |
| Risk-informed planning                       | Risk management                        | contingencies, issue modeling, controlled response planning      |
| Project-specific procurement                 | Procurement management                 | sourcing, long-lead planning, trade partner coordination         |
| Trade Partners                               | Procurement/delivery collaborators     | subcontractors, vendors, suppliers, specialty scopes             |
| Allies                                       | Trade Partner network members          | named partner network, preferred collaborators                   |

## Approved Military-Themed Terms

These terms are approved because they reinforce discipline without becoming theatrical:

- Command Center
- Operations Brief
- Comms Desk
- Command Staff
- Chain of Command
- Mission
- Deployment
- Rally Point
- Formation
- Briefing
- Handoff

## Approved Construction Terms

These terms are approved because they remain concrete and construction-native:

- Preconstruction
- Constructability
- Submittals
- RFIs
- Turnover
- Closeout
- Field coordination
- Procurement sequencing
- Scope control
- QA/QC

## Avoid Or Reframe

- Avoid `clients` when `Project Stakeholders` is the intended audience.
- Avoid `client partner` phrasing in new copy unless preserving a specific legacy reference.
- Avoid overly generic `partner` language when the audience is actually Trade Partners or Project Stakeholders.
- Avoid militarized language that adds style but removes clarity.

## Reference Pairs

- Project Stakeholders + Trade Partners
- Scope architecture + Sequence logic
- Budget clarity + Quality checkpoints
- Crew readiness + Briefings and handoffs
- Risk-informed planning + Project-specific procurement
